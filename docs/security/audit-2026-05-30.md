## Cross-Cutting Findings (appeared in multiple projects)

### 1. `torch.load()` without `weights_only=True` — RCE
**Projects:** chexpert-pathology-classifier, federated-credit-risk  
A malicious `.pt`/`.pkl` checkpoint executes arbitrary Python during deserialization. Both fixed to `torch.load(weights_only=True)`.

### 2. Unauthenticated API Endpoints on PHI/sensitive routes
**Projects:** chexpert, clinical-note-llmops, agentic-pipeline-healer, federated-credit-risk  
FastAPI endpoints accepting medical images, clinical notes, or triggering LLM pipelines had no auth. All fixed with `X-API-Key` header + `secrets.compare_digest`.

### 3. CORS Wildcard (`*`)
**Projects:** clinical-survival-analysis, chexpert, medical-imaging-ai, flight-ops, clinical-note-llmops, llm-safety-auditor  
All fixed to explicit `ALLOWED_ORIGINS` env var.

### 4. Stack Traces / Exception Details in HTTP Responses
**Projects:** clinical-survival-analysis, chexpert, flight-ops, medical-imaging-ai, clinical-note-llmops  
Internal exception strings leaking class names, file paths, and (in clinical-note-llmops) PHI fragments. All sanitized.

### 5. Missing Rate Limiting
**Projects:** clinical-survival-analysis, chexpert, llm-safety-auditor, flight-ops, kafka, clinical-note-llmops  
`slowapi` added to all relevant projects.

### 6. Model Weights Committed to Git
**Projects:** medical-imaging-ai (`model/xray_model.pth` 9.5MB tracked)  
Removed with `git rm --cached`. `.gitignore` hardened across all projects: `*.pt`, `*.pth`, `*.pkl`, `*.ckpt`, `*.joblib`.

## Project-Specific Highlights

### agentic-pipeline-healer — Subprocess Injection Bypass
Mock fix corpus contained `subprocess.run([sys.executable, '-m', 'pip', 'install', package_name])`. Validator only blocked `shell=True`. Real LLM path could trigger arbitrary package install from attacker-controlled DAG logs. Fixed: AST `Import` node inspection + regex now blocks all `subprocess`/`pickle`/`ctypes`/`importlib` references.

### sql-to-dag-compiler — Code Generation Injection (3 CRITICALs)
SQL embedded in Python triple-quoted strings: `"""` in SQL closes the string → arbitrary Python runs when Airflow imports the generated DAG. Table names used as Python identifiers without sanitization. dbt BashOperator built via f-string with no shell quoting. All three injection paths fixed with identifier regex allowlists + `shlex.quote()` + `_sanitize_sql_for_embedding()`.

### clinical-note-llmops — HIPAA Coverage Gap
Presidio regex fallback covers ~6 of 18 HIPAA identifiers. Silent degradation means PHI processed with incomplete scrubbing. Added `REQUIRE_PRESIDIO=true` startup guard — server refuses to run in unsafe mode. 4 identifier gaps remain (health plan beneficiary numbers, account numbers, vehicle IDs, device serials) — documented for custom recognizer implementation.

### federated-credit-risk — Model Poisoning + Pickle RCE
FedAvg accepted all client updates without validation — rogue institution could steer global credit model. L2-norm validation gate added. Pickle → `torch.load(weights_only=True)`. Gradient clipping added client-side.

### kafka-stream-feature-store — Redis Key Injection
`entity_id` concatenated directly into Redis keys — CRLF injection attempt possible (mitigated by redis-py binary safety but should never reach key). Regex validation `^[a-zA-Z0-9_\-\.]{1,128}$` added on all routes.

### Cross-Cutting Findings (private repos)

**CORS `*` + `allow_credentials=True` (CRITICAL — browser spec violation):**
adaptive-cognitive-rag, human-embryo-genetics, prompt-ops, ocean-sail-navigator, music-mood-engine, stellar-birth-chart, regulatory-compliance-copilot — all fixed to explicit origin allowlist or `allow_credentials=False`

**Path Traversal (CRITICAL):**
- `regulatory-compliance-copilot`: user-supplied `req.filepath` → `../../etc/passwd` readable — fixed with `Path.resolve()` + `BASE_DIR` prefix check
- `rag-compliance-intelligence`: user-supplied directory param on `/index` — fixed with `_resolve_safe_path()`

**Pickle RCE — no integrity verification:**
- `mlops-retraining-pipeline`: `pickle.loads()` on SQLite blobs with no signature → HMAC-SHA256 added to `register()`/`load()`
- `rag-compliance-intelligence`: `pickle.load()` on FAISS store → HMAC `.sig` file added
- `rag-memory-bank`: `pickle.load()` on index files → `Path.resolve()` + parent-dir confinement
- `music-mood-engine`: `pickle.load()` on model files → confinement guard added

**`torch.load()` RCE:**
- `healthcare-claims-anomaly`: missing `weights_only=True` → fixed
- `sign-language-translator`: explicit `weights_only=False` → fixed
